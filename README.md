# CLiGNet: Clinical Label-interaction Graph Network

**Multi-class Medical Specialty Classification from Clinical Transcriptions**

Built on the MTSamples clinical transcription dataset with 4,966 records across
40 medical specialties.

---

## Overview

This repository implements CLiGNet, a neural architecture that combines a
ClinicalBERT text encoder with a 2-layer Graph Convolutional Network (GCN)
operating on a specialty label graph. The model improves on standard one-versus-rest
BERT baselines by explicitly modeling structured relationships between clinical
specialties through semantic label graph reasoning.

The project corrects methodological flaws in prior work on this dataset
(data leakage from SMOTE applied before train/test splitting) and establishes
a reliable benchmark with proper statistical significance testing.

---

## Architecture

```
Clinical Transcription
        |
Bio_ClinicalBERT (sliding window, mean-pool [CLS])
        |
    doc_emb (768-dim)
        |
        +---------- Label Graph (40 nodes, semantic similarity + ICD-10 prior)
        |                   |
        |           GCNConv(768, 512) -> ReLU -> Dropout(0.3)
        |           GCNConv(512, 256) -> ReLU -> Dropout(0.3)
        |                   |
        |           node_embs (40 x 256)
        |                   |
        +--- Per-label Attention Gate (alpha_k for each of 40 labels) ---+
                            |
                    fused_k (40 x 256)
                            |
                    Linear(256, 1) per label -> sigmoid
                            |
                Focal BCE Loss (gamma=2.0, alpha=inverse frequency)
                            |
                Per-label Platt Scaling Calibration (val set)
```

---

## Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Imbalance | Focal loss (gamma=2.0) + inverse-frequency alpha | SMOTE causes data leakage when applied before split |
| Long docs | Sliding window (512 tok, stride 128, 4 chunks) | 34.9% of docs exceed 512 tokens |
| Label graph | Semantic similarity + ICD-10 prior | MTSamples is single-label; no true co-occurrence exists |
| Calibration | Per-label Platt scaling on val set | Rare classes have systematically miscalibrated probabilities |
| Significance | McNemar test + Bonferroni correction | Required for ACM venue submission |

## Baselines

| ID | Model | Key Config |
|----|-------|-----------|
| B1 | TF-IDF + One-vs-Rest LR | max_features=50k, ngram=(1,2), class_weight=balanced |
| B2 | TF-IDF + One-vs-Rest SVC | Platt calibration for probabilities |
| B3 | Bio_ClinicalBERT + OvR | Sliding window, linear head, focal loss |
| B4 | BioBERT-base + OvR | Same as B3 |
| B5 | InceptionXML | ACM SIGIR 2023 XMTC baseline |
| B6 | CLiGNet (no calibration) | Full CLiGNet, no Platt scaling |
| B7 | Clinical-Longformer + BR | 4096-token context, no GCN |
| B8 | CLiGNet Full | ClinicalBERT + GCN + fusion + focal + calibration |

## Ablation Study

| ID | Modification | Tests |
|----|-------------|-------|
| A1 | No GCN | Contribution of label graph structure |
| A2 | Standard BCE (no focal) | Contribution of focal loss |
| A3 | SMOTE (in-fold only) | Whether oversampling matches focal loss |
| A4 | No sliding window | Contribution of multi-chunk pooling |
| A5 | No Platt calibration | Contribution of per-label threshold calibration |

## Contributions

The paper claims four named contributions:

- **C1 — Benchmark Correction**: Document and correct data leakage in prior work; establish a clean MTSamples baseline.
- **C2 — Structured Label Modeling**: CLiGNet GCN + per-label attention + focal loss + Platt calibration.
- **C3 — Clinical Interpretability**: Per-label Integrated Gradients with expert rating protocol and UMLS signal recovery.
- **C4 — Failure Analysis**: Pairwise confusion (F1), rare class behaviour (F2), length vs. performance (F3), systematic error patterns (F4).

## Quick Start

### 1. Install dependencies

```bash
conda create -n clignet python=3.10 -y
conda activate clignet
pip install torch==2.1.0 --index-url https://download.pytorch.org/whl/cu118
pip install -r requirements.txt
pip install -e .
```

### 2. Download dataset

Place `mtsamples.csv` in `data/raw/`. See `data/raw/README.md`.

### 3. Preprocess

```bash
python scripts/preprocess.py --csv data/raw/mtsamples.csv --out data/processed/
```

### 4. Build label graph

```bash
python scripts/build_graph.py --processed data/processed/ --out data/processed/label_graph.pt
```

### 5. Train baselines B1 and B2

```bash
python scripts/train_baselines.py --processed data/processed/ --out results/baselines/
```

### 6. Train BERT baselines B3, B4, B7

```bash
python scripts/train_clignet.py --mode baseline --baseline-id B3 --out results/baselines/
python scripts/train_clignet.py --mode baseline --baseline-id B4 --out results/baselines/
python scripts/train_clignet.py --mode baseline --baseline-id B7 --out results/baselines/
```

### 7. Train CLiGNet B6 (no calibration) and B8 (full)

```bash
# B6: CLiGNet without calibration
python scripts/train_clignet.py --mode clignet --no-calibration --out results/clignet/

# B8: CLiGNet Full
python scripts/train_clignet.py --mode clignet --out results/clignet/
```

### 8. Evaluate all models

```bash
python scripts/evaluate.py --results results/ --processed data/processed/ --out results/
```

### 9. Run ablation study

```bash
python scripts/ablation.py --processed data/processed/ --graph data/processed/label_graph.pt \
                            --ablation A1 A2 A4 A5 --out results/ablations/
```

### 10. Interpretability and failure analysis

```bash
python scripts/interpret.py --processed data/processed/ \
                              --graph data/processed/label_graph.pt \
                              --model-dir results/clignet/B8/ \
                              --predictions results/clignet/B8/predictions.npz \
                              --out results/interpretability/
```
